EXPLICACIÓN 

Primero se cargó el archivo australian.dat con numpy. Este dataset contiene 690 registros y 15 columnas:
 14 columnas de entrada
 1 columna de salida

La última columna corresponde a la variable objetivo, que toma valores:
 0 = crédito rechazado
 1 = crédito aprobado

Luego se mezclaron los datos aleatoriamente y se dividieron en:
- 75% entrenamiento
- 25% prueba

Después se realizó la normalización de las variables de entrada. Este paso es importante porque las variables tienen diferentes escalas y el descenso por gradiente funciona mejor cuando los datos están normalizados. La normalización se hizo usando solamente los datos de entrenamiento, y luego esos mismos parámetros se aplicaron al conjunto de prueba. Esto evita fuga de información y ayuda a evaluar correctamente el rendimiento real
